## Guidelines

### Issues vs. Discussions

- **Issue** = actionable item (bug to fix, feature to build)
- **Discussion** = conversation (question, idea, feedback)

If unsure, start with a Discussion. It can be converted to an issue later.

### Response expectations

| Channel | Expected response time |
|---------|----------------------|
| Security reports | < 24 hours |
| Bug reports | < 3 days |
| Feature requests | < 7 days |
| Discussions / questions | < 7 days |
| Chat (Discord/Slack) | Best effort, no guarantee |
