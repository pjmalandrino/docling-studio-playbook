## Solo maintainer note

You might think "I don't need community docs, I'm solo." But:

- An onboarding guide attracts first contributors
- A public roadmap reduces "when will X be ready?" issues
- A triage process keeps your issue tracker manageable
- A governance model answers "who decides?" before it becomes a conflict

Start simple. Scale when needed.
